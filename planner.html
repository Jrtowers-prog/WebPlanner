Wireframing


Think of Wireframing as a documented plan for the layout of a website, in the same way that a blueprint acts for a house.

They are more useful than you might think, as having a pre-prepared layout allows you to pre-prepare as much as the code as possible, or atleast plan for it. This makes the development process itself much easier.

Take the above with a pinch of salt as I didn't use a wireframe when designing this site, although if it looks bad, maybe this will act as a lesson!



What You'll Need


To get started you're going to need what you see above, HTML, CSS and Javascript, JS for short.

HTML (or Hypertext Markup Language) is used to put together the building blocks of a website, titles, headers, paragraphs, footers etc.

CSS (or Cascading Style Sheets) is used to style the website, adding colour, font, shape etc.

The final and probably most difficult language to learn is Javascript, this is not essential for making a basic website, I haven't used it for this. However for making a modern Web2/Web3 application Javascript is essential as it adds for complex functionality.

If you want to learn any of these like I have, you can do so for free, or if you want to pay for extra features on Codecademy.
